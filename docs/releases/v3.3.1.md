# LLM Wiki 3.3.1

**Three-zone layout + full audit remediation**

Markdown-first multi-agent memory for solo developers. Zero cloud cost. Own your vault.

## Highlights

- **Three-zone architecture**: code + `knowledge/*` in the vault; runtime only under `$LLM_WIKI_STATE_ROOT/{run,logs,cache}`
- **Safer pipeline**: path containment, dry-run integrity, secret redaction, atomic compile lock
- **Install that works**: correct GitHub one-liners, OpenCode plugin update, Codex paths, **Claude settings merge** (backup-first)
- **Shipped examples**: public Evidence daily fixtures + sample knowledge notes
- **171 hermetic pytest tests**, CI with gitleaks + locked deps

## Install

```bash
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/Ekgardt/llm-wiki/main/install.sh | bash

# Windows (PowerShell)
irm https://raw.githubusercontent.com/Ekgardt/llm-wiki/main/install.ps1 | iex
```

Or:

```bash
git clone https://github.com/Ekgardt/llm-wiki.git
cd llm-wiki
# Windows: .\install.ps1   |   Unix: ./install.sh
```

## Upgrade from 3.3.0 / pre-three-zone

1. `git pull` (or re-run installer)
2. Ensure env:
   - `LLM_WIKI_ROOT` → vault path  
   - `LLM_WIKI_STATE_ROOT` → sibling `…/LLM-wiki-state` (or custom)
3. Runtime dirs are now `run/`, `logs/`, `cache/` (not `memory-state` / `memory-reports`)
4. Knowledge lives under `knowledge/` only (not root `memory/` or `wiki/`)
5. Claude Code users: re-run install (or `uv run python scripts/merge_claude_settings.py`) to refresh hooks
