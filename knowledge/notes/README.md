# knowledge/notes/ — Compiled durable pages

> Directory READMEs are exempt from the frontmatter requirement below.

Durable, citeable knowledge pages with OKF v0.1 frontmatter. Every page
carries at least `type:` and `title:`; durable claims also carry
`confidence`, `source_authority`, and a `## Evidence` block pointing at a
`knowledge/daily/<date>.md` block. (Wiki-derived pages may use `## Source`
instead of `## Evidence` — both are accepted.)

## Layout

Pages live **flat** as `<slug>.md` (the compile pipeline writes flat slugs).
Typed subdirectories (`concepts/`, `decisions/`, `patterns/`, …) are allowed
but currently unused — see `docs/ARCHITECTURE.md` for the taxonomy.

## What belongs here
- concepts, decisions, patterns, debugging notes, qa, workflows, gaps
- each page links outward via Obsidian-style `[[wikilinks]]`

## What does NOT belong here
- episodic session capture → `knowledge/daily/`
- raw sources → `knowledge/raw/`
- unprocessed staging → `knowledge/inbox/`
- per-project handoff → `knowledge/projects/<slug>/state.md`
- correction candidates → `knowledge/feedback/`

## Index + log
- `knowledge/index.md` — navigation map over knowledge pages (regenerated by
  `scripts/rebuild_memory_index.py`).
- `knowledge/log.md` — append-only editorial changelog.
