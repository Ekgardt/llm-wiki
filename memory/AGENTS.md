# Memory Subsystem ‚Äî AGENTS.md

Operating brief for any agent (Claude Code, scripted compiler, lint job) touching `memory/`.

## Scope
- You operate on `memory/` only. Do not write to `wiki/`, `raw/`, or `inbox/`.
- Promotion from `memory/` into `wiki/` is done via the `bridge-promote-insight` skill ‚Äî never inline.

## Layers
- `memory/daily/YYYY-MM-DD.md` ‚Äî append-only raw capture from session hooks (PreCompact, SessionEnd). Immutable log; do not rewrite.
- `memory/knowledge/{concepts,decisions,patterns,debugging,qa}/*.md` ‚Äî compiled durable pages.
- `memory/index.md` ‚Äî navigation over knowledge pages. Regenerated by `scripts/rebuild_memory_index.py`.
- `memory/log.md` ‚Äî append-only editorial changelog.
- `$LLM_WIKI_STATE_ROOT/memory-state/state.json` (default `$LLM_WIKI_STATE_ROOT/memory-state\state.json`) ‚Äî machine state for automation (last-flush, last-compile, per-file hashes, dedupe). Runtime lives OUTSIDE the vault so Obsidian and git don't track ephemeral churn.
- `$LLM_WIKI_STATE_ROOT/memory-reports/` ‚Äî lint reports, compile stdout/stderr logs, SessionStart debug dumps. Also runtime, not versioned content.

## Page conventions
Every knowledge page:
- `# Title`
- `One-sentence summary: ...`
- `## Lesson` (or `## Decision`, `## Symptom / Cause / Resolution`, etc.)
- `## Evidence` ‚Äî pointer to `memory/daily/<date>.md` and `[HH:MM:SS]` block
- `## Related` ‚Äî wikilinks to sibling memory pages and (if promoted) the wiki counterpart

## What to lift into knowledge
Lift an item only if it is:
- reusable beyond the session it came from,
- not already derivable from code, config, or `wiki/`,
- specific enough to act on ("when X, do Y because Z").

Do NOT lift:
- status updates, task progress,
- restatements of file paths or code structure,
- speculation not validated by use,
- summaries of `raw/` / `inbox/` material (those belong in `wiki/`).

## Categories
- **concepts/** ‚Äî noun-shaped mental models.
- **decisions/** ‚Äî dated choice + alternatives rejected + reason. Immutable; supersede, don't edit.
- **patterns/** ‚Äî verb-shaped recurring approach.
- **debugging/** ‚Äî symptom ‚Üí cause ‚Üí resolution.
- **qa/** ‚Äî settled answer to a non-obvious recurring question.

Tiebreak: patterns > concepts; debugging > qa.

## Automation contract
- `scripts/flush_memory.py` ‚Äî classifies a session transcript into one of three tiers (Phase 0.5):
  - **FLUSH_MAJOR** ‚Äî contains decisions/lessons/non-obvious commands ‚Üí append structured summary to daily log AND trigger compile after `MEMORY_COMPILE_AFTER_HOUR` (default 18:00).
  - **FLUSH_MINOR** ‚Äî contains only gotchas/debug-notes/open-questions ‚Üí append to daily log but do NOT trigger compile.
  - **FLUSH_OK** ‚Äî pure status/progress chatter ‚Üí skip entirely, record dedupe only.
  The classifier emits the tier token as the first line, followed by the structured block (skipped for OK). Triggered by PreCompact + SessionEnd (never PostCompact as primary path).
- `scripts/compile_memory.py` ‚Äî compiles changed daily logs into knowledge pages using a SKEPTICAL prompt that requires VERIFY-BEFORE-WRITE (every Evidence citation must be re-read against the cited `[HH:MM:SS]` block before the page is written). The compiler emits two sentinels: `COMPILE_DONE: <n> page(s) touched` and `COMPILE_AUDIT: verified <a> citations; <b> dedup checks; <c> stubs skipped; <d> contradictions handled; <e> pages rejected`. The audit line is stored in `state.json::last_compile_audit`. Incremental: skips daily logs whose SHA-256 matches the last compile (via `compiled_daily_hashes` in `state.json`).
- `scripts/codex_memory.py daily-log` ‚Äî Phase 0.5 anti-pollution: when `--transcript` is empty (the default for Codex CLI), records only an activity heartbeat in `state.json::codex_heartbeats` and does NOT write a stub block into `memory/daily/`. Pass `--transcript <path>` to take the full flush path; pass `--force-stub` to restore legacy stub-writing behavior.
- `scripts/user_prompt_capture.py` ‚Äî non-LLM, ms-fast hook for UserPromptSubmit. Appends a one-line breadcrumb `- [HH:MM:SS] prompt | <session> | <slug> <preview>` to today's daily log. Rate-limited per (slug, prompt-hash) within 30s. Skips prompts <5 chars and sessions inside the vault itself.
- `scripts/post_tool_capture.py` ‚Äî non-LLM, ms-fast hook for PostToolUse. Same shape as the prompt capture but filtered to SIGNIFICANT_TOOLS (Edit, Write, MultiEdit, NotebookEdit, Bash). Bash commands <8 chars are skipped (cd/pwd noise).
- `scripts/migrate_to_okf.py` ‚Äî one-shot migration to OKF v0.1 frontmatter. Idempotent: re-running on conformant pages is a no-op. Infers `type` from path, extracts `title` from H1, `description` from "One-sentence summary:" line. Use `--apply` to write, `--scope {wiki,memory,skills,rules,projects,all}` to limit.
- `scripts/cognee_sync.py` ‚Äî OPTIONAL semantic-layer bridge. Echoes wiki + memory/knowledge pages into a Cognee graph for entity extraction + relationship queries. Graceful degradation: exits 0 with a one-line notice if `cognee` not installed, Ollama not running, or env not configured. Never fails the parent pipeline. See `docs/SETUP-COGNEE.md`.
- `scripts/query_memory.py` ‚Äî answers from memory only; can file-back a Q&A page with `--file-back`.
- `scripts/lint_memory.py` ‚Äî twelve checks across `memory/` and `wiki/`: broken wikilinks, orphan pages, orphan daily logs, stale compiled pages, missing backlinks, sparse pages (<200 words by default), and ‚Äî opt-in ‚Äî LLM-judged contradictions (1-7, the original structural set); plus OKF conformance checks 8-12 (missing_frontmatter, missing_required_type, missing_sources_section, invalid_supersede_chain, orphan_gaps). Use `--scope memory|wiki|all`, `--contradictions`, `--sparse-words N`.
- Recursion guard: all hook wrappers (`session_end_capture.py`, `precompact_capture.py`) short-circuit when `CLAUDE_INVOKED_BY` is set, so SDK sub-sessions spawned by `flush_memory`/`compile_memory` don't re-trigger the memory pipeline on themselves.

## Multi-tool integration
The memory pipeline is reachable from three surfaces:
- **Claude Code** ‚Äî `~/.claude/settings.json` registers SessionStart / UserPromptSubmit / PreCompact / PostToolUse / SessionEnd hooks that call the scripts above.
- **OpenCode** ‚Äî `~/.config/opencode/plugins/llm-wiki-memory.js` plugin subscribes to `session.created`, `session.idle`, `experimental.session.compacting`, `tool.execute.after` events and shells out to the same scripts. Autoloads at startup. OpenCode has no native UserPromptSubmit equivalent ‚Äî tool capture + idle cover the same surface.
- **Codex CLI** ‚Äî `scripts/codex_memory.py` wrapper provides `project-state`, `daily-log`, `state-path`, `lookup-tier` subcommands. Manual invocation at end of a Codex session: `python $LLM_WIKI_ROOT/scripts\codex_memory.py daily-log --cwd <your-projects-dir>/<project> --reason codex-session-end`.

## Boundary with `wiki/`
Promote from `memory/` to `wiki/` only when the insight generalizes beyond this repo's internal workflow. See `memory/operating-model.md` for the full rule.
